# PROJECT SPECIFICATION MODE (Мод создания проектной документации)

Этот режим предназначен для поэтапного и командного создания проектной документации с разделением на несколько ключевых документов и расширенными секциями для сбора максимального количества данных.

## Workflow (по шагам)
1. **business-requirements.md** (бизнес-аналитик):
   - Требования, user stories, acceptance criteria
   - Бизнес-процессы (BPMN, flowcharts), value stream mapping
   - Анализ рынка, конкурентный анализ (SWOT, feature matrix)
   - Customer journey, pain points, сегментация
   - Критерии успеха, ограничения, риски
   - AI-генерация вопросов для уточнения
2. **product-vision-roadmap.md** (продукт-менеджер):
   - Product vision, roadmap, value/effort matrix
   - KPI/OKR, customer feedback, гипотезы
   - Анализ рынка, discovery, customer journey
3. **architecture-spec.md** (архитектор):
   - Архитектурные решения, схемы, диаграммы
   - Интеграции, паттерны, технические/нефункциональные требования
   - Риски, mitigation, зависимости
   - AI-генерация вопросов для уточнения
4. **ai-summary.md** (AI-ассистент):
   - Краткое резюме, выявленные риски, рекомендации
   - Ссылки на связанные решения, паттерны, knowledge packages
5. **epic-brief.md** (на каждый эпик):
   - Описание эпика, цели, AI-summary
   - User stories, сценарии, критерии успеха (DoD)
   - Варианты реализации, альтернативы, story map
   - Value/effort matrix, риски, зависимости
   - Визуализации (mindmap, journey map, wireframes)
   - Галерея, lessons learned
6. **brainstorming.md** (команда + AI):
   - Сбор идей, mindmap, обсуждение, голосование
   - AI-генерация альтернативных сценариев, плюсы/минусы
7. **features.md** (PM + команда):
   - Список функций: реализовано, в планах, альтернативы
   - AI-предложения новых функций, приоритеты
8. **release-plan.md** (PM):
   - План релизов, критерии готовности, зависимости
   - Для каждого релиза — список функций, задач, целей
9. **design-spec.md** (дизайнер):
   - Wireframes, user flow, moodboard, UI-kit
   - Accessibility, анимации, AI-вопросы для дизайнера
10. **effort-estimate.md** (PM + команда):
    - Таблица трудозатрат, роли, story points, дедлайны
11. **integration-plan.md** (архитектор/PM):
    - Интеграции, API, внешние сервисы, риски, AI-вопросы

## Как наполнять и согласовывать
- Каждый документ наполняется своей ролью (BA, PM, Architect, Designer, QA, DevOps, AI).
- После заполнения — проводится согласование (peer review, AI-проверка полноты, голосование).
- Все документы связаны ссылками, AI-ассистент помогает выявлять пробелы, предлагает альтернативы, автоматизирует обновление связей и summary.
- Для каждого эпика, релиза, функции — отдельный документ, галерея, визуализация, lessons learned.

## Расширенный шаблон projectBrief.md
```markdown
# Project Brief: <Название проекта>

## 1. Бизнес-требования
- Описание целей, задач, value proposition
- User stories (пример):
  - Как <роль>, я хочу <действие>, чтобы <ценность>
- Acceptance criteria (пример):
  - Пользователь может зарегистрироваться через email
- Бизнес-процессы (BPMN, flowcharts)
- Value stream mapping (пример):
  - [Диаграмма](./img/value-stream.png)
- Анализ рынка, конкурентный анализ (SWOT, feature matrix)
- Customer journey, pain points, сегментация
- Критерии успеха, ограничения, риски
- AI-генерация вопросов для уточнения:
  - Какие внешние системы должны быть интегрированы?
  - Какие метрики считаются успешными?

## 2. Product Vision & Roadmap
- Product vision, ключевые метрики успеха (KPI, OKR)
- Roadmap (этапы, релизы, приоритеты)
- Value/effort matrix, customer feedback
- AI-генерация гипотез, сценариев развития

## 3. Архитектура и интеграции
- Архитектурные решения, схемы, диаграммы
- Интеграции, паттерны, технические ограничения
- Технические и нефункциональные требования (security, performance, scalability)
- Риски, mitigation, зависимости
- AI-генерация вопросов для уточнения:
  - Какие SLA/SLI/SLO критичны?
  - Какие требования к безопасности?

## 4. AI-summary
- Краткое резюме, выявленные риски, рекомендации
- Ссылки на связанные решения, паттерны, knowledge packages

## 5. Чек-лист полноты
- [ ] Требования собраны и формализованы (BA)
- [ ] User stories и acceptance criteria проработаны
- [ ] Бизнес-процессы и value stream описаны
- [ ] Анализ рынка и конкурентов проведён
- [ ] Customer journey и pain points выявлены
- [ ] Критерии успеха, ограничения, риски зафиксированы
- [ ] Product vision/roadmap сформированы (PM)
- [ ] Value/effort matrix и feedback учтены
- [ ] Архитектура, интеграции, схемы описаны (Architect)
- [ ] Технические/нефункциональные требования учтены
- [ ] AI-summary сгенерирован и проверен
- [ ] Документация опубликована и связана с core-файлами Memory Bank
```

## Примеры для каждой секции
- **User story:** Как маркетолог, я хочу видеть аналитику по кампаниям, чтобы оптимизировать бюджет.
- **Acceptance criteria:** В отчёте отображаются все активные кампании за выбранный период.
- **SWOT:** Strengths — уникальный AI-движок; Weaknesses — мало интеграций.
- **Value stream mapping:** [Диаграмма](./img/value-stream.png)
- **Customer journey:** Регистрация → onboarding → первый запуск → получение value → upsell.
- **Техническое требование:** Время отклика API < 200мс при 1000 RPS.
- **AI-вопрос:** Какие внешние API критичны для MVP?
